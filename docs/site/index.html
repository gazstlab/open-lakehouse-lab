Documentação do Open Lakehouse Lab

Uma página estática para estudar a arquitetura local com kind, Kubernetes, Airflow, MinIO, Apache Iceberg, Apache Polaris, DuckDB, dbt, Prometheus e Grafana. O projeto é educacional, reproduzível localmente e focado na comunidade brasileira de engenharia de dados.

Executar exemplos Seguir lições Criar pipeline próprio
M1 Subir a plataforma kind, MinIO, Polaris e Airflow rodando localmente.
M2 Publicar dados Raw Fixture Parquet gravada no bucket lakehouse.
M3 Transformar com dbt DuckDB processando Raw, Silver e Gold.
M4 Orquestrar no Airflow DAGs criando pods efêmeros no cluster kind.

Arquitetura

O caminho padrão valida a coluna dorsal do projeto antes de introduzir adapters concretos de fonte. DuckDB é a engine SQL de processamento; MinIO armazena dados persistentes; Polaris registra tabelas Iceberg; Airflow orquestra workloads em pods Kubernetes.

Estrutura do projeto

Cada diretório representa uma parte do laboratório. A ideia é que você consiga navegar pelo código como se fosse um mapa da plataforma.

airflow/

Projeto Airflow criado com Astro CLI, DAGs principais e DAGs didáticas.

dbt/

Projeto dbt Core com modelos Raw, staging, Silver, intermediate e marts.

k8s/

Manifests Kubernetes, Helm values, RBAC e recursos locais da plataforma.

docker/

Dockerfiles das imagens locais usadas por Airflow e workloads dbt.

ingestion/

Espaço para adapters de fonte e utilitários de ingestão plugáveis.

metadata/

Artefatos previstos para catálogo, qualidade, freshness e execuções.

docs/

Documentação, runbooks, lições, troubleshooting e esta página estática.

scripts/

Automações locais, validadores e explicações didáticas dos atalhos.

.github/

Workflows de convenções, qualidade e automações do GitHub Actions.

Consulta SQL com DuckDB

dbt usa DuckDB como engine de transformação. Para explorar o arquivo local criado por dbt: 

duckdb dbt/target/open_lakehouse_lab.duckdb
show schemas;
select * from main_raw_sources.generic_raw_contract limit 10;
select * from main_staging.stg_raw_source_events limit 10;
Feche DuckDB CLI, DuckDB UI ou extensões do VS Code conectadas ao mesmo arquivo antes de rodar dbt. DuckDB permite apenas um processo de escrita por arquivo.

Customização de pipelines

O usuário pode usar o exemplo padrão como referência e criar o próprio pipeline adicionando dados Raw, modelos dbt e, quando necessário, DAGs Airflow.

Objetivo Arquivos
Ler Raw Parquet dbt/models/raw_sources/generic_raw_contract.sql
Normalizar nomes e tipos dbt/models/staging/*.sql
Criar tabelas Silver dbt/models/silver/*.sql
Criar tabelas Gold dbt/models/intermediate/*.sql e dbt/models/marts/*.sql
Estudar Airflow airflow/dags/lab_*.py

Contrato Raw canônico

s3://lakehouse/raw/source=<source>/dataset=<dataset>/ingestion_date=YYYY-MM-DD/*.parquet

Colunas técnicas mínimas:

source
dataset
ingestion_date
loaded_at
record_hash
raw_payload

Airflow

DAG principal

open_lakehouse_lab_daily e o exemplo padrão estável. Ela publica fixture Raw, roda dbt para Raw/staging/Silver, testa Silver, roda intermediate/marts e testa Gold. 

make trigger-airflow-dbt
kubectl -n data-platform exec deployment/airflow-scheduler -- \
  airflow dags list-runs open_lakehouse_lab_daily

DAGs didáticas

lab_kubernetes_pod_operator e lab_params_and_retries existem para estudar recursos do Airflow sem quebrar o pipeline principal. 

kubectl -n data-platform exec deployment/airflow-scheduler -- \
  airflow dags trigger lab_kubernetes_pod_operator

kubectl -n data-platform exec deployment/airflow-scheduler -- \
  airflow dags trigger lab_params_and_retries

Troubleshooting

Cluster kind já existe 
kind get clusters
make cluster-status

Se quiser recomeçar do zero, rode make cluster-delete e depois make cluster-create.

Job da fixture Raw expira por timeout 
kubectl -n data-platform get pods -l job-name=dbt-publish-raw-fixture
kubectl -n data-platform logs job/dbt-publish-raw-fixture
kubectl -n data-platform describe job dbt-publish-raw-fixture

Confirme que MinIO, Polaris e a imagem dbt local estão prontos.

UI do Airflow retorna erro de CSRF

Feche o port-forward, limpe cookies de localhost:8080 ou use janela anônima, e abra novamente.

make port-forward-airflow
Banco DuckDB está bloqueado

Feche DuckDB CLI, DuckDB UI ou extensões conectadas ao mesmo arquivo antes de rodar dbt.
