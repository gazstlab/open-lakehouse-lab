Documentacao do Open Lakehouse Lab

Uma pagina estatica para estudar a arquitetura local com kind, Kubernetes, Airflow, MinIO, Apache Iceberg, Apache Polaris, DuckDB, dbt, Prometheus e Grafana. O projeto e educacional, reproduzivel localmente e focado na comunidade brasileira de engenharia de dados.

Arquitetura

O caminho padrao valida a coluna dorsal do projeto antes de introduzir adapters concretos de fonte. DuckDB e a engine SQL de processamento; MinIO armazena dados persistentes; Polaris registra tabelas Iceberg; Airflow orquestra workloads em pods Kubernetes.

As credenciais padrao sao locais: MinIO usa minioadmin / minioadmin123; Airflow usa admin / admin; Polaris usa variaveis locais de desenvolvimento.

Consulta SQL com DuckDB

dbt usa DuckDB como engine de transformacao. Para explorar o arquivo local criado por dbt: 

duckdb dbt/target/open_lakehouse_lab.duckdb
show schemas;
select * from main_raw_sources.generic_raw_contract limit 10;
select * from main_staging.stg_raw_source_events limit 10;
Feche DuckDB CLI, DuckDB UI ou extensoes do VS Code conectadas ao mesmo arquivo antes de rodar dbt. DuckDB permite apenas um processo de escrita por arquivo.

Customizacao de pipelines

O usuario pode usar o exemplo padrao como referencia e criar o proprio pipeline adicionando dados Raw, modelos dbt e, quando necessario, DAGs Airflow.

Objetivo Arquivos
Ler Raw Parquet dbt/models/raw_sources/generic_raw_contract.sql
Normalizar nomes e tipos dbt/models/staging/*.sql
Criar tabelas Silver dbt/models/silver/*.sql
Criar tabelas Gold dbt/models/intermediate/*.sql e dbt/models/marts/*.sql
Estudar Airflow airflow/dags/lab_*.py

Contrato Raw canonico

s3://lakehouse/raw/source=<source>/dataset=<dataset>/ingestion_date=YYYY-MM-DD/*.parquet

Colunas tecnicas minimas:

source
dataset
ingestion_date
loaded_at
record_hash
raw_payload

Airflow

DAG principal

open_lakehouse_lab_daily e o exemplo padrao estavel. Ela publica fixture Raw, roda dbt para Raw/staging/Silver, testa Silver, roda intermediate/marts e testa Gold. 

make trigger-airflow-dbt
kubectl -n data-platform exec deployment/airflow-scheduler -- \
  airflow dags list-runs open_lakehouse_lab_daily

DAGs didaticas

lab_kubernetes_pod_operator e lab_params_and_retries existem para estudar recursos do Airflow sem quebrar o pipeline principal. 

kubectl -n data-platform exec deployment/airflow-scheduler -- \
  airflow dags trigger lab_kubernetes_pod_operator

kubectl -n data-platform exec deployment/airflow-scheduler -- \
  airflow dags trigger lab_params_and_retries

Troubleshooting

Cluster kind ja existe 
kind get clusters
make cluster-status

Se quiser recomecar do zero, rode make cluster-delete e depois make cluster-create.

Job da fixture Raw expira por timeout 
kubectl -n data-platform get pods -l job-name=dbt-publish-raw-fixture
kubectl -n data-platform logs job/dbt-publish-raw-fixture
kubectl -n data-platform describe job dbt-publish-raw-fixture

Confirme que MinIO, Polaris e a imagem dbt local estao prontos.

UI do Airflow retorna erro de CSRF

Feche o port-forward, limpe cookies de localhost:8080 ou use janela anonima, e abra novamente.

make port-forward-airflow
Banco DuckDB esta bloqueado

Feche DuckDB CLI, DuckDB UI ou extensoes conectadas ao mesmo arquivo antes de rodar dbt.
